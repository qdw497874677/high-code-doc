# 技术设计文档规范

## 文档分层

技术设计文档按抽象程度分为两层：

| 阶段 | 目的 | 时机 |
|------|------|------|
| [[概要设计]] | 对齐方向，评审决策 | 需求评审后 |
| [[详细设计]] | 指导实现，补充细节 | 开发前 |

## 流程

```mermaid
graph LR
    A[需求分析] --> B[概要设计]
    B --> C[概要评审]
    C --> D[详细设计]
    D --> E[开发实现]
```

## 核心文件

- [[概要设计]] - 概要设计规则与内容
- [[详细设计]] - 详细设计规则与内容
- [[通用规则]] - 写作禁忌、评审规则、迭代规则
